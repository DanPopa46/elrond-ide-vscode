If you wish to change the configured folder or the download mirror, please review the extension's settings.

In order to build smart contracts to WASM bytecode, a subset of LLVM is necessary. More specifically, clang, llc and wasm-ld.

Furthermore, a variant of the Elrond node is required for debugging smart contracts. This variant, called node-debug, enables a HTTP endpoint which is then used by the Visual Studio Code extension to deploy smart contracts and execute exported functions.

Please note that in order to deploy and run on a testnet, you need to separately download and run an Elrond node.
